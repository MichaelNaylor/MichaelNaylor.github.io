Programming Unit


About this unit


In the first year during this unit I will be uning the Java programming language. Java is an object oriented language that can be used on almost any operating system that contains a run time enviroment for it. Due to its usage of a run time enviroment Java programmes can be compiled once and then run anywhere, instead of being re-compiled for each machine that they need to run on.


During the first few weeks of this unit we are using a grapical IDE called Processing. Processing has its own built in library which can help to make writing code faster as a lot of code is already written for you and ready to use within the IDE. Processing is an open source program that is free to use. The IDE can be downloaded from processing.org. The website also contains a lot of tutorials showing how to use the IDE, help with the Java language and it has a forum through which users can talk to each other and ask for help.


After the first few weeks when we are more familiar with the core concepts of the Java language we will move onto a widely used professional IDE called Eclipse. Eclipse is used by Java developers all over the world and like Processing it is open source and free to use. Eclipse can be downloaded from eclipse.org . Unlike Processing Eclipse does not come with its own extensive library, however it is possible to inport the processing library and many others into your Eclipse projects.

Procedurally drawn monster


In this task I needed to draw a monster face using procedures. This was achived by creating a procedure to draw each element of the face and then a final procedure to position each of these elements. This procedure can then be called in setup or draw and passes position and size paramaters to draw the face. Pong Game


In this task I was asked to create a pong style game. This was to consist of a movable bat on the left hand side of the screen controlled by the player and a ball that would move continously around the screen. Hitting any of the top, bottom and right hand sides will cause the ball to bounce back however the ball will leave the screen via the left side unless it hits the bat. The objectve of the game is to keep the ball in the playing area for as long as possible. The ball speeds up each time it hits the bat to make the game harder, and each hit increases the score by 1. You can view the code for this project in my GitHub repository.
